Запуск
----------

python3 fact_extractor.py 
-----------------------------------

Скрипт протестирован на правилах из папки Rules
-------------------------------

Скрипт 
* проходится по обязательным элементам правила (obligatory, key, dummy)
* анализирует тип состовляющей, вершины и морфологические теги элементов из правила и элементов из json
* сопоставляет такие элементы. Находит наиболее подходящие пары. (Возможна ситуация, когда на место пары элемента из правила претендуют более одного элемента из json. В таком случае запоминаются все претенденты для проверки синтаксических связей. НО! Такое вариант еще не оттестирован! Скорее всего нуждается в доработке)
* проверяет, присутствуют ли указанные синтаксические связи из правила в json.
* выводит извлеченный факт

Если на каком-то этапе находится несоответствие, то выводится соответствующее предупреждение (не найден обязательный тег, не найдена связь..). Так как не реализовано считывает файлов и json-ов циклом, нет выхода из функций, если на каком-то этапе возвращается false (т.е. не найден обязательный тег, не найдена связь..).

Скрипт анализирует только обязательные элементы. В дальнейшем возможно нужно добавить анализ опциональных.
